# AI 学习助手 - 部署与使用说明

## 环境要求

- Python 3.9+
- Windows / macOS / Linux 均可

## 快速开始

### 1. 安装依赖

打开终端，进入项目目录：

```bash
cd ai-learning-assistant
pip install -r requirements.txt
```

### 2. 配置 API Key

用任意文本编辑器打开 `.env` 文件，修改以下配置：

```env
LLM_API_KEY=KEY
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-chat
```

| 字段 | 说明 |
|------|------|
| `LLM_API_KEY` | 必填，大模型 API 密钥 |
| `LLM_BASE_URL` | API 地址，默认 DeepSeek；如用 OpenAI 兼容接口可改 |
| `LLM_MODEL` | 模型名称，按需修改 |

### 3. 启动

**Windows：** 双击 `start.bat`

**命令行启动（任意系统）：**

```bash
python backend/app.py
```

启动后访问：http://127.0.0.1:5000

## 功能说明

- **知识问答**：上传文档后，基于文档内容进行 AI 对话
- **知识测评**：选择文档自动生成测评题，实时评判掌握度
- **学习进度**：查看知识点掌握度雷达图和学习统计
- **学习报告**：AI 自动生成周学习报告，支持 Markdown 下载

## 文件结构

```
ai-learning-assistant/
├── .env                    ← API Key 配置（必须修改）
├── config.py               ← 全局配置
├── requirements.txt        ← Python 依赖
├── start.bat               ← Windows 一键启动
├── backend/                ← 后端服务
├── frontend/               ← 前端页面
├── data/                   ← 运行时数据（自动生成）
│   ├── uploads/            ← 上传的文档
│   ├── vector_db/          ← 向量数据库
│   ├── reports/            ← 生成的报告
│   └── learning.db         ← 学习记录数据库
└── embedding_model/        ← 本地 Embedding 模型（离线可用）
```

## 常见问题

**Q: 启动报错 `ModuleNotFoundError`**
A: 依赖没装全，重新执行 `pip install -r requirements.txt`

**Q: 页面打开空白**
A: 确认终端显示 `Backend starting...` 且无报错；检查端口 5000 是否被占用

**Q: 上传文档后问答无响应**
A: 检查 `.env` 中 `LLM_API_KEY` 是否正确填写

**Q: 端口冲突**
A: 在 `.env` 中添加 `FLASK_PORT=5001` 换一个端口

**Q: Embedding 模型加载失败**
A: 项目已内置本地模型（`embedding_model/`），正常情况无需联网下载；如目录缺失会自动从 HuggingFace 下载
